Ce que ça fait

01

Trouve le rapport XML

Cherche dans les chemins Maven standard. Si aucun rapport trouvé, vérifie la config pom.xml et peut lancer PiTest lui-même.

02

Lit le code source et les tests

Pour chaque classe mutée, lit le fichier source ET les tests associés avant d'analyser. Détecte le style de test utilisé (AAA, BDD, librairies).

03

Analyse en 4 sections actionnables

Mutations critiques avec score de risque, faux positifs à exclure du pom.xml, snippets de tests à ajouter, bilan et suggestions de configuration.

Maxi-prompt

system-prompt.txt 
Analyse le projet Maven dans lequel tu te trouves.

Localise le rapport PiTest XML dans target/pit-reports/, lis le code source
et les tests des classes mutées, puis produis une analyse en 4 sections :

🐛 1. Mutations critiques
Les mutants survivants qui impactent la logique métier : classe/méthode/ligne,
ce que le mutant modifie concrètement, l'impact en production, pourquoi le
test actuel ne le tue pas. Score de risque 🔴/🟠/🟡.

🧟 2. Faux positifs écartés
Les mutants sans valeur réelle (toString, logs, code généré) avec la config
d'exclusion exacte à ajouter dans le pom.xml.

🛡️ 3. Tests à renforcer
Pour chaque mutant critique, un test complet dans le style déjà utilisé dans
le projet. Commenter l'assertion clé avec : // ← tue le mutant [TYPE] ligne X.

📊 4. Bilan
Score actuel, estimation après correction des critiques, zones sans couverture,
observations sur la qualité de la suite, suggestions de config PiTest.

Installer le skill

01

Télécharger le fichier

Clique sur "Télécharger le skill" ci-dessus pour récupérer le fichier pitest-analyzer.skill.

02

Ouvrir les paramètres Claude

Dans Claude.ai → Settings → Skills → Install from file. Sélectionner le fichier téléchargé.

03

Utiliser depuis la codebase

Ouvrir Claude depuis la racine du projet Maven. Le skill se déclenche automatiquement dès qu'on mentionne PiTest ou mutation testing.
